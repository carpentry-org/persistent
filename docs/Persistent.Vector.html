push-back-owned

defn

(Fn [a, (Persistent.Vector a)] (Persistent.Vector a)) 

                        (push-back-owned value vec)

Append, consuming the vector instead of borrowing it.

When the caller hands over ownership (moves the vector in rather than passing a reference), the common tail-only append mutates the tail in place — no copy, no allocation — because tails are never shared between versions. Use this in build/churn loops that discard the previous version; it is a transient-style fast path with the same result as push-back. The rarer tail-flush case falls back to a copying push.
