Design

Design overview for persistent.

Purpose

persistent provides macro-generated immutable data structures for Carp, implemented with structural sharing on top of rc.

Included Structures

  • Persistent List (Persistent.define-list)
  • Persistent Queue (Persistent.define-queue)
  • Persistent Trie Map (Persistent.define-trie)
  • Persistent Deque (Persistent.define-deque)
  • Persistent Ordered Map (Persistent.define-ord-map)
  • Persistent Ordered Set (Persistent.define-ord-set)
  • Persistent Min-Heap (Persistent.define-heap)
  • Persistent Hash Map (Persistent.define-hash-map)
  • Persistent Hash Set (Persistent.define-hash-set)
  • Persistent Vector (Persistent.define-vector)

Why Macros

persistent needs to generate full concrete type families per instantiation, not only polymorphic functions:

  • concrete wrapper + internal recursive node types
  • Rc.define integration for each concrete node type
  • explicit per-type namespaces and specialized codegen

Architecture

  • updates are path-copy: only touched nodes are rebuilt
  • untouched branches are shared through Rc
  • all major traversals are iterative for deep-input stack safety

Representation Notes

  • list/queue/deque: shared persistent list nodes
  • trie/hash/vector: trie path routing and path-copy updates
  • ord-map/set: unbalanced persistent BST
  • heap: persistent skew-heap merge core

Semantics

  • most structures provide structural =
  • heap intentionally omits structural =
  • every family provides reduce, each, and to-array

Safety Model

Inherited from rc:

  • single-threaded usage model
  • non-atomic reference counters
  • no cycle collection
  • iterative payload teardown in rc drain queue
