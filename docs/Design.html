Design

How persistent is built.

This is for people curious about the internals. If you just want to use the library, the README and per-function docs cover the user surface.

What 'persistent' means here

Each collection update returns a new collection that shares most of its memory with the old one. The old version stays valid. That's it. Updates allocate only along the path that changed; everything else is reused by pointer. Rc (reference counting) keeps the shared parts alive until the last version that points at them is gone.

Why a macro generator

Each collection is actually a small family of types: a public wrapper, one or more internal recursive node types, reference-counted handles for those nodes, and a module of operations. Rc.define itself is a generator that runs per node type, so the whole stack has to be specialized at expansion time. Generating it through Persistent.define-* gives you stable namespaces (IntList, StrMap, …) and direct calls with no runtime dispatch.

Collections at a glance

  • List: singly-linked. Cheap prepend/head/tail.
  • Queue: two-list FIFO with amortized O(1) ends.
  • Deque: two-list double-ended queue.
  • Trie: keyed by (Array K), supports prefix queries.
  • Ordered map / set: persistent AVL tree (self-balancing BST).
  • Heap: persistent skew min-heap. Everything is merge.
  • Hash map / set: trie keyed by hash digits (depth 8, radix 16).
  • Vector: trie keyed by index digits (depth 7, radix 32).

Equality

ptr-eq compares backing roots and counts; cheap, useful for short-circuiting work. = is structural for everything except heap (multiset equality is caller-defined).

Iteration

Every collection exposes reduce, each, and to-array. Map-likes also expose keys and values; the trie additionally exposes reduce-values.

Stack safety

Algorithms that would naively recurse on the C stack are written iteratively: list drop-n, queue/deque reversals, trie path rebuilds, ord-map path rebuilds, heap merge spine. Deep inputs are bounded by heap memory rather than C stack depth.

Inherited from rc

  • single-threaded; non-atomic counters
  • no cycle collection
  • iterative payload teardown (drain queue) so dropping a deeply nested structure doesn't recurse on the C stack
